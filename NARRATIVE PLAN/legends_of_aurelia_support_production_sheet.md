# Legends of Aurelia: The Rift  
## Support Writing Production Sheet

This document converts the current support web into a **production-facing implementation sheet** for writing, pacing, and system integration.

It is meant to answer:
- which support chains should be written first,
- when each chain becomes available,
- suggested support point gates,
- and which A-rank romantic chains should lock out other romances.

---

# 1. Recommended Support System Rules

## Rank Structure
- **C Rank** = first meaningful breakthrough
- **B Rank** = deeper trust / ideological conflict / emotional complication
- **A Rank** = final resolution / emotional culmination

## Recommended Point Thresholds
Use these as a baseline for implementation:

- **Unlock C:** 40 support points
- **Unlock B:** 90 support points
- **Unlock A:** 160 support points

## Suggested Point Gain Sources
- +3 points: end map if both units deployed
- +2 points: adjacent for 3+ turns in a map
- +2 points: one unit heals / rescues / shields the other
- +1 point: shared camp event or exploration interaction
- +4 points: special scripted joint event

## Recommended Chapter Gating
To prevent supports from firing too early:

- **C ranks:** available as soon as both units are recruited
- **B ranks:** usually gated to the chapter after recruitment, or after one major shared story beat
- **A ranks:** gated to late midgame or later unless explicitly a growth-unit chain

---

# 2. Romance Lock Rule Recommendation

## Recommended Romance System
Only **one romantic A-rank** can be finalized per character.

If the Avatar / Commander confirms one romantic A-support, all other romantic A-supports involving the Avatar should be locked at B-rank maximum.

For non-avatar romances, you can choose one of two models:

### Option A — Strict Lockout
A character can only complete one romantic A-support total.

### Option B — Narrative Lockout Only at Ending
Multiple romantic A-supports can be viewed, but only one paired ending is selected in the epilogue.

## Best Fit for This Game
**Recommended: Option A for the Avatar, Option B for side characters.**

This keeps the Commander’s route emotionally cleaner while giving you flexibility for optional writing elsewhere.

# 7. Implementation Notes

## Recommended Support Menu Sorting
Sort chains in this order:
1. Avatar supports
2. Kaelen / Liora / Nyx / Darian / Branik core cast supports
3. faction-linked supports
4. late-game supports
5. growth-unit and flavor supports

## Recommended Visibility Rule
Hide supports for unrecruited units entirely.

## Recommended Conversation Unlock Rule
If both units have enough points for the next rank, show an exclamation mark in camp / Greyspire hub.

## Recommended A-Rank Timing Rule
Do not allow most A-ranks before **Level 11**.
Do not allow tragic late-game A-ranks involving Hadrien before **Level 17**.

---

# 8. Final Note

This sheet is designed to keep the support system from:
- firing too early,
- flattening emotional pacing,
- or resolving relationships before the campaign earns them.

The overall goal is:
- early C-ranks build premise,
- midgame B-ranks deepen the army,
- late A-ranks make the endgame hurt more.

That is the current recommended production structure.
